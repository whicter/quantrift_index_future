# NQ/ES 价差策略 — 详细设计文档

## 策略原理

NQ（纳斯达克100）与 ES（标普500）短期高度相关。
当 NQ 的 1H 涨跌幅显著高于 ES（或反之），这种短期"背离"通常在 1-2 小时内修复。

## 使用收益率价差而非价格价差的原因

- 价格价差（NQ - β×ES）：β 不稳定（1.7~7.9），长期漂移，协整检验失败
- 收益率价差（r_NQ - r_ES）：天然平稳，无需估计 β，统计性质更好

## 信号计算

```python
r_NQ(t) = NQ(t) / NQ(t-1) - 1      # 1H 涨跌幅
r_ES(t) = ES(t) / ES(t-1) - 1

r_spread(t) = r_NQ(t) - r_ES(t)    # 收益率价差

# 滚动 Z-score（240h = 10天 窗口）
Z(t) = (r_spread(t) - mean_240) / std_240
```

## 持仓方向

| Z-score | 含义 | 操作 |
|---------|------|------|
| Z > +2.0 | NQ 涨太多，ES 相对便宜 | 空 MNQ + 多 MES |
| Z < -2.0 | ES 涨太多，NQ 相对便宜 | 多 MNQ + 空 MES |
| |Z| < 0.5 | 价差回归 | 平仓 |

## 对冲比例（美元中性）

```
n_MES = β × MNQ_mult / MES_mult = 4.76 × 2 / 5 ≈ 2

实际使用：1 MNQ + 2 MES
```

P&L 计算（以多NQ空ES为例，pos=+1）：
```
pnl = (p_NQ_exit - p_NQ_entry) × $2 × 1   # MNQ 多头
    + (p_ES_entry - p_ES_exit) × $5 × 2   # MES 空头
```

## 参数

| 参数 | 值 | 说明 |
|------|---|------|
| z_entry | 2.0 | 入场阈值（回测最优）|
| z_exit | 0.5 | 出场阈值（价差回归）|
| z_stop | 3.5 | 止损阈值（价差扩大）|
| window | 240h | Z-score 滚动窗口（10天）|
| max_bars | 8 | 最长持仓 8h |
| commission | $12 | 3 legs × $4（估计）|

## 执行架构

实盘需要：
1. 整点 bar 收盘后计算 Z-score
2. 若 |Z| > 2.0，立即同时下两条腿市价单
3. 后续每小时检查 Z-score，决定是否平仓
4. 平仓时同样两腿同时平

关键挑战：
- 两腿执行有延迟（几百毫秒），可能造成 leg risk
- 整点后价差可能瞬间回归，导致入场即亏损
- 需要实测验证

## 风险与局限

1. **非协整**：价格价差长期漂移，不保证长期有效
2. **β 漂移**：NQ/ES 关系在 2025 年关税冲击期间严重偏离
3. **执行风险**：双腿同步执行在实盘中不可完全保证
4. **容量限制**：微型合约，流动性充足，但大资金无法扩容
