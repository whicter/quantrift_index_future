# 策略说明文档

**策略名称**: Confluence 多指标趋势跟踪 + ES 均值回归（两套独立系统）
**标的**: NQ（MNQ）+ ES（MES），CME 微型期货
**周期**: 趋势 1H / 4H / 1D | MR 1H

---

## 系统一：趋势跟踪引擎（主引擎）

**文件**: `live_engine.py` + `strategy.py` + `indicators.py`
**品种**: NQ（全部 TF）+ ES（4H / 1D）
**clientId**: 20，pm2 进程: `ib-bot`

### 策略逻辑概述

核心思想：**多个独立指标同方向共振时才入场**，单一指标不足以触发信号。

每根 Bar 收盘时，计算各指标方向，累加"牛市分" (bullScore) 和"熊市分" (bearScore)。
当牛市分 ≥ `min_score` 且无拥挤/矛盾信号时，开多；反之开空。

### 参与评分的指标

#### 1. UT Bot（趋势跟踪止损）
- 基于 ATR 的自适应追踪止损线
- 价格突破上方止损线 → +1 牛；突破下方 → +1 熊
- 参数：`ut_key`（ATR 倍数，越大越不敏感）

#### 2. SSL Channel（趋势通道）
- 基于 EMA 的上下轨，判断价格在通道内的位置
- 收盘价在上轨上方 → +1 牛；下轨下方 → +1 熊
- 参数：`ssl_len`（EMA 周期）

#### 3. MACD（动量）
- 标准 MACD（12/26/9）
- MACD 线 > 信号线 → +1 牛；反之 → +1 熊

#### 4. RSI（动量过滤）
- RSI > 50 → +1 牛；RSI < 50 → +1 熊

#### 5. ADX（趋势强度过滤，可选）
- ADX < `adx_threshold` 时过滤掉信号（震荡市不入场）
- 参数：`use_adx`, `adx_threshold`

#### 6. 成交量过滤（可选）
- 当前成交量 > 均量 × `vol_mult` 时才允许入场

#### 7. BBMC（Bollinger Band + MACD Color）
- 布林带中轨方向作为大趋势过滤
- `bbmc_dir` ≥ 0 → 允许做多；< 0 → 允许做空（当 `use_bbmc_dir=True`）

### 入场条件（多头示例）

```
bullScore >= min_score          # 足够多的指标方向一致
bearScore <= conflict_threshold # 反向指标不能太多
ADX >= adx_threshold            # 趋势足够强（如启用）
bbmc_dir >= 0                   # 大趋势向上（如启用）
```

### 退出条件

- **止损**：`utTS` 动态追踪止损线（UT Bot trailing stop）
- **分批止盈（Staged TP）**：
  - TP1：ATR × 1.0，平掉 34% 仓位
  - TP2：ATR × 2.0，平掉剩余仓位
- **反向信号翻转**：持多出现强空头信号（bearScore ≥ reversal_score）→ 平多开空

## 系统二：ES 均值回归引擎（MR 引擎）

**文件夹**: `es_mr/`
**品种**: ES（MES）1H only，**次季合约**（deferred，跳过当期前月）
**clientId**: 21，pm2 进程: `ib-bot-mr`

### 策略逻辑

ES 在 1H 周期内频繁震荡，价格极端偏离后倾向快速修复（平均 2 小时）。

**只做多**（ES 2024-2026 牛市，空头 MR 胜率仅 38%，长期负期望）

### 入场条件（三个全中）

| 信号 | 条件 |
|------|------|
| RSI | < 28（超卖） |
| 布林带 | 价格 ≤ 下轨（20期，2.0σ） |
| VWAP | 价格 < VWAP − 2.0 × ATR |
| 市场过滤 | ADX < 25（非趋势行情） |

### 退出条件（优先级从高到低）

1. 止损：`entry − 1.0 × ATR`
2. 时间止损：持仓超 8 根 1H bar
3. ATR 止盈：`entry + 2.0 × ATR`（R:R = 2:1）
4. VWAP 止盈：价格回到 VWAP

### 回测结果（2024-06-10 ~ 2026-06-08）

| 指标 | 值 |
|------|----|
| 总收益率 | **+72.5%** |
| 年化收益率 | 25.6% |
| Sharpe Ratio | **0.723** |
| Max Drawdown | **-13.4%** |
| Profit Factor | **2.585** |
| 胜率 | 53.3% |
| 总笔数 | 15（约 1 笔/月） |

## 仓位管理（两套引擎共用逻辑）

### 定仓公式

```
风险金额 = 账户净值 × risk_pct%
止损金额 = ATR × sl_mult × 合约乘数
手数     = 风险金额 / 止损金额（最小 1 手）
```

### 两种风险模式（趋势引擎）

| 净值 | 模式 | NQ risk_pct | ES risk_pct |
|------|------|-------------|-------------|
| < $60,000 | 统一模式 | 所有周期 1.0% | 所有周期 1.0% |
| ≥ $60,000 | Pyramid 模式 | 1H:0.72% / 4H:2.4% / 1D:6.0% | 4H:1.5% / 1D:4.5% |

**各净值档位典型手数：**

| 品种/周期 | 止损/手 | $32k（统一1%）| $60k（Pyramid）| $100k（Pyramid）| $200k（Pyramid）|
|----------|--------|-------------|--------------|----------------|----------------|
| MNQ 1H | $240 | 1手 | 2手 | 3手 | 6手 |
| MNQ 4H | $600 | 1手 | 2手 | 4手 | 8手 |
| MNQ 1D | $1,200 | 1手 | 3手 | 5手 | 10手 |
| MES 4H | $375 | 1手 | 2手 | 4手 | 8手 |
| MES 1D | $750 | 1手 | 4手 | 6手 | 12手 |
| **NQ最大净仓** | | **3手** | **7手** | **12手** | **24手** |

> 关键：risk_pct 的增幅必须大于 ATR 的增幅，否则高周期 ATR 大反而手数少，产生倒置金字塔。NQ 1D ATR 是 1H 的 5 倍，故 1D risk_pct（6.0%）须显著高于 1H（0.72%）。

> ⚠️ **小账户风险失真**：公式结果 < 1 手时强制取 1 手最小值。$32k 下 NQ 1D 实际风险 = $1,200 / $32k = **3.75%**（意图 1%）。5 TF 同时触发止损最大损失 ≈ -$3,165（-9.9%）。不会爆仓，但需有心理准备，不要 panic 干预。此系统设计目标为 $100k+ 账户。

> MR 引擎：ATR 动态定仓，固定 1% 风险/笔。手数 = max(1, round(净值×1% / (ATR×1.0×$5)))。$32k→4手 / $60k→8手 / $100k→13手。

### 三周期仓位叠加（趋势引擎）

三个周期各自独立信号、独立持仓，净仓 = 三 TF signed_contracts 之和：

- 同向叠加：多周期顺势时自动加仓
- 反向对冲：低周期翻转时自动降风险敞口

## 三、信号增强计划（Pattern + 背离检测）

> **背景**：2025-04-09 NQ 暴涨事件复盘发现，当前策略所有退出信号仅靠 UT Bot 追踪止损，
> 无任何图形形态或反转结构识别能力。以下为计划引入的增强信号，全部用于**持仓退出优化**，
> 不改变入场逻辑。

### 3.1 Squeeze Momentum Indicator（SMI）背离

**指标说明**：LazyBear SMI，结合布林带（BB）+ 肯特纳通道（KC）检测价格挤压状态，
动量柱（线性回归动量）判断突破方向和强度。

**背离定义**：
- **底背离**：价格创 N 根新低，但 SMI 动量柱未创新低（绿柱更高）→ 下行动量衰竭
- **顶背离**：价格创 N 根新高，但 SMI 动量柱未创新高（红柱更低）→ 上行动量衰竭

**使用方式**：
- 持有空仓时出现底背离 → 触发止盈平仓
- 持有多仓时出现顶背离 → 触发止盈平仓
- 多时间级别（1H / 4H）同时背离 → 信号更强

**参数**：`smi_len`（BB/KC 周期，默认 20），`smi_lookback`（背离回望窗口，默认 5 根 bar）

---

### 3.2 RSI 背离检测

**背离定义**：
- **底背离**：价格创新低，RSI 未创新低 → 多头动量增强
- **顶背离**：价格创新高，RSI 未创新高 → 空头动量增强

**使用方式**：与 SMI 背离联合确认，两者同时触发信号更可靠。

**参数**：`divergence_lookback`（回望窗口，默认 5 根 bar）

---

### 3.3 4H 插针（Pin Bar）识别

**识别条件**（任意一个）：
- 锤子线：下影线 > 实体 × 2 且下影线 > 上影线 × 2（底部反转）
- 射击之星：上影线 > 实体 × 2 且上影线 > 下影线 × 2（顶部反转）

**使用方式**：
- 4H 锤子线出现在近期低点区域 → 持有空仓时触发止盈
- 4H 射击之星出现在近期高点区域 → 持有多仓时触发止盈

**参数**：`pin_wick_ratio`（影线/实体倍数，默认 2.0）

---

### 3.4 双底 / 双顶形态

**识别条件**：
- 双底：近 N 根 bar 内出现两个相近的低点（价差 < ATR × 0.5），第二个低点后价格强力收回
- 双顶：近 N 根 bar 内出现两个相近的高点，第二个高点后价格快速回落

**使用方式**：
- 1H 双底确认（第二低点后收盘高于第一低点）→ 持有空仓时触发止盈
- 与 SMI/RSI 背离联合使用，减少误判

**参数**：`double_bottom_lookback`（扫描窗口，默认 20 根 bar），`double_bottom_atr_tol`（价差容忍度，默认 0.5 ATR）

---

### 信号优先级与组合规则

| 信号 | 单独触发 | 联合确认 |
|------|---------|---------|
| SMI 底背离 | 触发止盈 | + RSI 背离 → 强触发 |
| 4H 锤子线 | 触发止盈 | + SMI 背离 → 强触发 |
| 双底 | 辅助确认 | 需 SMI 或 RSI 同步背离 |

> **实现位置**：`indicators.py`（计算信号列）+ `strategy.py`（退出逻辑判断）
> **影响范围**：仅退出逻辑，入场条件不变，不影响开仓频率和 Sharpe 下限

---

## 执行原理（回放法）

引擎不实时订阅 tick，每根 Bar 收盘后：

1. 从 IB 拉取最新历史数据
2. 运行完整回测，读取策略最终持仓状态
3. 对比 IB 实际持仓与状态文件，计算差额
4. 按差额下单（市价单）

优点：实盘信号与回测信号完全一致。
